En el siglo 21, la tecnología es una parte integral de nuestra vida diaria. Sin embargo, muchas tareas todavía se realizan manualmente, en especial las administrativas. Esto es particularmente cierto para las universidades que deben gestionar una gran cantidad de constancias, cada una con sus datos individuales. Estos datos deben ser ingresados ​​por el personal administrativo para ser almacenados en formularios de papel y luego ser enviados a los estudiantes. Este procedimiento, aunque clásico, suele generar costes de gestión considerables, pérdidas de tiempo para quienes quieren recibir sus documentos rápidamente y es proclive a errores humanos. Parecería que existe la necesidad de una herramienta de automatización que asegure una administración eficiente. Por lo tanto, el objetivo de este proyecto es crear una aplicación de escritorio para resolver este problema, impulsado por el reciente interés de la Universidad Tecnológica de Uruguay. Se recopilaron los requerimientos de varias secretarías para crear un sistema que satisficiera sus necesidades. Hay una amplia gama de expectativas para este proyecto, pero el núcleo sigue siendo el mismo: dejar atrás el trabajo a mano por el click.

El producto a ser desarrollado fue denominado “Gestor de Solicitudes de Constancias - GSC” 
GSC administra la solicitud, emisión y registro de constancias y reclamos, ordenando así la gestión de las mismas en la carrera LTI de los ITR de UTEC. Dichas constancias son: Constancia de Presencial (Presencial común, presencial prueba), Constancia de transporte, Constancia de Estudiante activo, Constancia de exámenes y Constancia de créditos (VME, UTEC innova, optativas)
El sistema cuenta con tres perfiles de usuarios: analista, docente, y estudiante. También, es importante señalar que este sistema tiene como objetivo ser utilizado para que los estudiantes logren solicitar diferentes tipos de constancias y las analistas de la carrera se las emitan a través del mismo, a su vez registrando los diferentes eventos que determinarán la validez de la misma.
Adicionalmente, el sistema brinda un dashboard con información sobre los diferentes eventos, incluyendo las personas registradas en los mismos y notificaciones sobre las diferentes solicitudes. 

Para el desarrollo del sistema fue seleccionada una combinación de tecnologías vistas en la carrera. En el área de programación, el modelo de arquitectura está basado en un modelo Cliente – Servidor, utilizando EJB’S Y JPA. Se implementará una Interfaz Swing comunicándose con los EJB remotos que contienen la lógica, que a la vez se comunican con los DAO para el acceso a los datos. En cuanto a Base de Datos, se trabajará con Oracle XE.
